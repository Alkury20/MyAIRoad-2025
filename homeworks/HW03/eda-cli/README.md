# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

CLI предоставляет две основные команды для анализа CSV-файлов:

### 1. Команда `overview` – Краткий обзор датасета

Выводит базовую информацию о CSV-файле: количество строк, столбцов, типы данных и краткую статистику по колонкам.

```bash
uv run eda-cli overview data/example.csv
```

**Доступные параметры:**

- `--sep` – разделитель в CSV (по умолчанию `,`)
- `--encoding` – кодировка файла (по умолчанию `utf-8`)

**Пример с параметрами:**

```bash
uv run eda-cli overview data/example.csv --sep ";" --encoding "cp1251"
```

---

### 2. Команда `report` – Полный EDA-отчёт

Генерирует комплексный отчёт с визуализациями и статистикой в указанном каталоге.

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

**Доступные параметры:**

- `--out-dir` – каталог для сохранения отчёта (по умолчанию `reports`)
- `--sep` – разделитель в CSV (по умолчанию `,`)
- `--encoding` – кодировка файла (по умолчанию `utf-8`)
- `--title` – заголовок отчёта в `report.md` (по умолчанию `,`)
- `--max-hist-columns` – максимальное количество числовых колонок для построения гистограмм (по умолчанию `6`)
- `--top-k-categories` – количество top-значений для категориальных признаков (по умолчанию `5`)

**Новые параметры и их влияние на отчёт:**

- **`--title`**: Задаёт заголовок отчёта (первая строка в `report.md`). Полезно для идентификации отчётов по разным датасетам.
  
- **`--max-hist-columns`**: Ограничивает количество числовых колонок, для которых будут построены гистограммы. Используйте меньшее значение для больших датасетов, чтобы не создавать слишком много графиков.

- **`--top-k-categories`**: Определяет, сколько наиболее частых значений будет выведено для каждого категориального признака в файлах `top_categories/*.csv`. Увеличьте значение для более детального анализа категорий.

**Примеры использования с новыми опциями:**

```bash
# Отчёт с кастомным заголовком и ограничением гистограмм
uv run eda-cli report data/example.csv \
  --out-dir reports_example \
  --title "Анализ пользовательских данных" \
  --max-hist-columns 10

# Отчёт с большим количеством top-значений для категориальных признаков
uv run eda-cli report data/example.csv \
  --out-dir reports_detailed \
  --top-k-categories 10 \
  --title "Детальный анализ датасета"

# Полный пример со всеми новыми параметрами
uv run eda-cli report data/example.csv \
  --out-dir my_reports \
  --title "Комплексный EDA-отчёт" \
  --max-hist-columns 8 \
  --top-k-categories 7 \
  --sep "," \
  --encoding "utf-8"
```

**Результат выполнения команды `report`:**

В каталоге `reports/` (или указанном через `--out-dir`) появятся:

- `report.md` – основной отчёт в Markdown с указанным заголовком
- `summary.csv` – таблица по колонкам
- `missing.csv` – пропуски по колонкам
- `correlation.csv` – корреляционная матрица (если есть числовые признаки)
- `top_categories/*.csv` – top-k категорий по строковым признакам (количество определяется `--top-k-categories`)
- `hist_*.png` – гистограммы числовых колонок (количество ограничено `--max-hist-columns`)
- `missing_matrix.png` – визуализация пропусков
- `correlation_heatmap.png` – тепловая карта корреляций
