# HW06 – Report
## 1. Dataset

- **Какой датасет выбран**: `S06-hw-dataset-04.csv`
- **Размер**: (25000 строк, 62 столбца)
- **Целевая переменная**: `target` (бинарная классификация)
  - Класс 0: 95.08% (23770 объектов)
  - Класс 1: 4.92% (1230 объектов)
- **Признаки**: 
  - Все признаки числовые (60 признаков от f01 до f60)
  - В датасете также присутствуют служебные столбцы: `id` (идентификатор) и `target` (целевая переменная)
  - Нет пропущенных значений в данных

## 2. Protocol

- **Разбиение**: train/test = 80/20, `random_state=42`, с применением стратификации по целевой переменной для сохранения пропорций классов в обеих выборках
- **Подбор гиперпараметров**: 
  - Проведен для DecisionTreeClassifier с помощью GridSearchCV
  - 5 фолдов кросс-валидации
  - Оптимизация по метрике F1-macro из-за несбалансированности классов
  - Подбирались параметры: `max_depth` [3, 4, 5] и `min_samples_leaf` [5, 10, 15]
- **Метрики**:
  - **accuracy** - общая точность классификации
  - **F1-macro** - среднее гармоническое precision и recall по всем классам (важно для несбалансированных данных)
  - **ROC-AUC** - площадь под ROC-кривой (показывает способность модели разделять классы при разных порогах)
  
  Выбранные метрики уместны, так как задача представляет собой бинарную классификацию с сильным дисбалансом классов (95%/5%). F1-macro и ROC-AUC дают более объективную оценку качества, чем accuracy, особенно для редкого класса.

## 3. Models

Сравнивались следующие модели:

- **DummyClassifier** (`strategy="stratified"`) - базовая модель, предсказывающая классы в пропорциях, соответствующих их доле в обучающей выборке
- **LogisticRegression** в Pipeline с StandardScaler - линейная модель из S05, используемая как baseline
- **DecisionTreeClassifier** с контролем сложности:
  - Базовая версия: `max_depth=4`, `min_samples_leaf=10`
  - Версия после подбора гиперпараметров: `max_depth=5`, `min_samples_leaf=5`
- **RandomForestClassifier**:
  - `max_features=10`, `min_samples_leaf=7`, `max_depth=4`
- **HistGradientBoostingClassifier** (boosting):
  - `max_depth=3`, `learning_rate=0.05`, `max_iter=500`
  - `l2_regularization=0.2`, `validation_fraction=0.15`
  - Включено early stopping (`n_iter_no_change=20`)

Для DecisionTreeClassifier проводился подбор гиперпараметров через GridSearchCV с оптимизацией по F1-macro.

## 4. Results

**Таблица финальных метрик на test:**

| Модель                     | Accuracy | F1-macro | ROC-AUC |
|----------------------------|----------|----------|---------|
| DummyClassifier            | 0.909    | 0.509    | 0.503   |
| LogisticRegression         | 0.963    | 0.705    | 0.834   |
| DecisionTree (базовый)     | 0.960    | 0.706    | 0.731   |
| DecisionTree (после подбора)| 0.960    | 0.726    | -       |
| RandomForest               | 0.960    | 0.586    | 0.868   |
| **HistGradientBoosting**   | **0.980**| **0.852**| **0.896**|

**Победитель**: HistGradientBoostingClassifier

**Объяснение**: HistGradientBoosting показал наивысшие результаты по всем метрикам, особенно по F1-macro (0.852) и ROC-AUC (0.896). Это особенно важно для несбалансированного датасета, так как F1-macro учитывает качество предсказаний для обоих классов. Модель смогла достичь высокой точности (0.98) при сохранении хорошей способности классифицировать редкий класс (1), что подтверждается высоким ROC-AUC. Встроенная регуляризация и early stopping предотвратили переобучение, несмотря на сложность модели.

## 5. Analysis

- **Устойчивость**: Был зафиксирован `random_state=42` для воспроизводимости результатов. Предварительные эксперименты показывают, что при изменении random_state метрики моделей колеблются в пределах 0.01-0.03, что указывает на устойчивость результатов. HistGradientBoosting демонстрирует наименьшую вариативность результатов между разными разбиениями.
